What are Snapshots?

Snapshots track changes in source data over time - perfect for SCD Type 2.

# snapshots/snapshot_products.sql
{% snapshot snapshot_products %}

{{
    config(
        target_schema='snapshots',
        unique_key='product_id',
        strategy='timestamp',
        updated_at='updated_at',
    )
}}

select * from {{ ref('stg_products') }}

{% endsnapshot %}

TypeDescriptionUse Case

🎯 Snapshot Strategies

# Timestamp strategy (when updated_at exists)
{{
    config(
        strategy='timestamp',
        updated_at='updated_at',
    )
}}

# Checksum strategy (when no timestamp)
{{
    config(
        strategy='check',
        check_cols=['price', 'stock', 'name'],
    )
}}

Query Snapshot Data

select * from {{ ref('snapshot_products') }}
where dbt_valid_to is null  -- Current state only

⚡ Airflow Integration

DBT DAG Example

# dags/dbt_daily_pipeline.py
from airflow import DAG
from airflow.operators.bash import BashOperator
from datetime import datetime, timedelta

default_args = {
    'owner': 'data_team',
    'retries': 1,
    'retry_delay': timedelta(minutes=15),
}

with DAG(
    'dbt_daily_pipeline',
    default_args=default_args,
    schedule_interval='0 2 * * *',
    catchup=False,
) as dag:

    run_dbt = BashOperator(
        task_id='run_dbt',
        bash_command='cd /path/to/dbt && source venv/bin/activate && dbt run',
    )

    test_dbt = BashOperator(
        task_id='test_dbt',
        bash_command='cd /path/to/dbt && dbt test',
    )

    notify = BashOperator(
        task_id='notify',
        bash_command='curl -X POST $SLACK_WEBHOOK -d "DBT completed"',
    )

    run_dbt >> test_dbt >> notify

Key Configuration

  • schedule_interval='0 2 * * *' - Run at 2 AM daily
  • catchup=False - Don't backfill missed runs
  • retries=1 - Retry once on failure
