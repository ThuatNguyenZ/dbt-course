🏗️ Data Layers Explained

📦 RAW LAYER (S3)

Data nguyên gốc từ API, không qua transform. Lưu ở S3 dạng JSON/CSV.

🔧 STAGING LAYER (DBT)

Cleaning, renaming, type casting. 1:1 với source tables. Ví dụ: stg_shopee__orders

⚡ INTERMEDIATE LAYER (DBT)

Business logic, joins, aggregations. Reusable components cho marts.

📊 MARTS LAYER (DBT → PostgreSQL)

Final tables phục vụ reporting/analytics. Ví dụ: fct_daily_sales

🎯 Tạo First DBT Model

Step 1: Tạo Staging Model cho Shopee Orders

-- models/staging/stg_shopee__orders.sql

{{
    config(
        materialized='view',
        schema='staging',
        tags=['shopee', 'orders', 'staging']
    )
}}

with source as (
    select *
    from {{ source('shopee', 'orders_raw') }}
),

renamed as (
    select
        -- Identifiers
        order_id::varchar as order_id,
        buyer_id::varchar as customer_id,
        seller_id::varchar as seller_id,

        -- Timestamps
        order_date::timestamp as order_at,
        paid_date::timestamp as paid_at,
        shipped_date::timestamp as shipped_at,
        completed_date::timestamp as completed_at,

        -- Amounts
        total_amount::numeric as gross_amount,
        discount_amount::numeric as discount,
        shipping_fee::numeric as shipping_fee,
        voucher_amount::numeric as voucher_discount,

        -- Status
        order_status as status,
        payment_method as payment_type,

        -- Metadata
        _loaded_at as loaded_at
    from source
)

select * from renamed

Step 2: Define Sources

# models/staging/schema.yml

version: 2

sources:
  - name: shopee
    description: "Raw data từ Shopee API"
    schema: raw_data
    tables:
      - name: orders_raw
        description: "Orders raw data"
        columns:
          - name: order_id
            description: "Unique order identifier"
            tests:
              - unique
              - not_null
          - name: total_amount
            description: "Total order amount"
            tests:
              - not_null
      - name: products_raw
        description: "Products raw data"

  - name: tiktok
    description: "Raw data từ TikTokShop API"
    schema: raw_data
    tables:
      - name: orders_raw
        description: "TikTok orders raw data"
      - name: products_raw
        description: "TikTok products raw data"

💡 Pro Tips

  • ✓ Đặt tên file theo pattern: stg_{source}__{entity}.sql
  • ✓ Dùng CTE (with clause) để organize code
  • ✓ Type casting ngay từ staging layer
  • ✓ Thêm tests vào columns quan trọng

📚 Sources & Seeds

Sources vs Seeds - Khi nào dùng gì?

Feature Sources Seeds
Definition Raw tables trong database CSV files trong project
Use Case Data từ API, ETL pipelines Static reference data
Examples orders_raw, products_raw exchange_rates, mappings
Location PostgreSQL schema seeds/ folder

Seed Example: Exchange Rates

# seeds/exchange_rates.csv
currency,date,rate_to_vnd
VND,2024-01-01,1
USD,2024-01-01,24500
CNY,2024-01-01,3400
THB,2024-01-01,700

# Update rates trong dbt_project.yml
seeds:
  dbt-shopee-tiktok:
    exchange_rates:
      +schema: reference_data

🚀 Commands

# Load seeds vào database
dbt seed

# Run models
dbt run

# Run tests
dbt test

# Run specific model
dbt run --models stg_shopee__orders

# Run với tags
dbt run --select tag:shopee

📝 Hands-on Exercise

Bài tập thực hành

Tạo hoàn chỉnh staging layer cho Shopee và TikTok data

📋 Yêu cầu

  1. Tạo project DBT mới: dbt init dbt-ecommerce
  2. Configure connection tới PostgreSQL RDS
  3. Tạo sources.yml cho shopee và tiktok sources
  4. Tạo 4 staging models:
    • stg_shopee__orders.sql
    • stg_shopee__products.sql
    • stg_tiktok__orders.sql
    • stg_tiktok__products.sql
  5. Thêm tests (unique, not_null) cho các columns quan trọng
  6. Chạy dbt run và dbt test
