🖥️ EC2 Setup

Launch EC2 Instance

# AWS CLI
aws ec2 run-instances \
    --image-id ami-0c55b159cbfafe1f0 \
    --instance-type t3.medium \
    --key-name your-key \
    --security-group-ids sg-your-sg \
    --tag-specifications 'ResourceType=instance,Tags=[{Key=Name,Value=dbt-runner}]'

Security Group Rules

TypePortSource
SSH22Your IP
Custom TCP8080Your IP (DBT docs)

Install Dependencies

# SSH vào EC2
ssh -i your-key.pem ec2-user@your-ec2-ip

# Install Docker
sudo yum install -y docker
sudo service docker start
sudo usermod -aG docker ec2-user

# Install Python
sudo yum install -y python3.11 python3.11-pip

🐳 Docker Containerization

Dockerfile

# Dockerfile
FROM python:3.11-slim

WORKDIR /dbt

COPY requirements.txt .
RUN pip install --no-cache-dir -r requirements.txt

COPY . .

ENV DBT_PROFILES_DIR=/dbt/profiles

ENTRYPOINT ["dbt"]
CMD ["run"]

Build & Run

# Build image
docker build -t dbt-shopee-tiktok .

# Run container
docker run --rm \
    -v $(pwd)/profiles:/dbt/profiles:ro \
    -e AWS_ACCESS_KEY_ID=$AWS_ACCESS_KEY_ID \
    -e AWS_SECRET_ACCESS_KEY=$AWS_SECRET_ACCESS_KEY \
    dbt-shopee-tiktok dbt run

# Run with docker-compose
docker-compose up -d
docker-compose logs -f dbt

Benefits of Docker

  • ✓ Consistent environment across dev/prod
  • ✓ Easy deployment
  • ✓ Isolation from host system

⚡ Near-Real-Time Architecture

Batch vs Streaming

AspectBatchStreaming
LatencyHours/DaysSeconds/Minutes
Complexity✓ Low✗ High
Cost✓ Low✗ High
Use CaseDaily reportsReal-time dashboards

Near-Real-Time Options

# Option 1: Hourly Batch (Recommended for most cases)
# Airflow schedule: 0 * * * * (every hour)

# Option 2: Kinesis → Lambda → DBT
# API → Kinesis Stream → Lambda (process) → S3 → DBT

# Option 3: Change Data Capture (CDC)
# Debezium → Kafka → DBT (with dbt-cloud)

⚠️ Consider Before Going Real-Time

  • Do you REALLY need sub-minute latency?
  • 10x more complex than batch
  • Start with hourly batch → optimize later
