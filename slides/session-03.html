🔄 Data Deduplication

Tại sao cần deduplication?

❌ Vấn đề

  • API pull nhiều lần → duplicate records
  • Retry failed jobs → inserted twice
  • Data quality issues từ source
  • Late-arriving data

✓ Giải pháp

  • Row numbering với window functions
  • Pick latest record per business key
  • Handle late-arriving data
  • Idempotent pipeline

Deduplication Pattern

-- models/intermediate/int_orders__deduped.sql

{{
    config(
        materialized='table',
        schema='intermediate',
        tags=['orders', 'dedup']
    )
}}

with source as (
    -- Union all sources
    select *, 'shopee' as source from {{ ref('stg_shopee__orders') }}
    union all
    select *, 'tiktok' as source from {{ ref('stg_tiktok__orders') }}
),

ranked as (
    select
        *,
        row_number() over (
            partition by order_id
            order by loaded_at desc,
            order by _dbt_loaded_at desc
        ) as rn
    from source
)

-- Select only the latest record per order_id
select
    order_id,
    customer_id,
    seller_id,
    source,
    order_at,
    paid_at,
    shipped_at,
    completed_at,
    gross_amount,
    discount,
    shipping_fee,
    voucher_discount,
    status,
    payment_type,
    loaded_at
from ranked
where rn = 1

⚠️ Lưu ý quan trọng

  • Chọn loaded_at DESC để lấy record mới nhất
  • Partition by business key (order_id), không phải surrogate key
  • Include source column để debug khi có issues

🔗 Joins & Data Enrichment

Enriching Orders với Customer Data

-- models/intermediate/int_orders__enriched.sql

{{
    config(
        materialized='table',
        schema='intermediate',
        tags=['orders', 'enriched']
    )
}}

with orders as (
    select * from {{ ref('int_orders__deduped') }}
),

customers as (
    select * from {{ ref('stg_shopee__customers') }}
    union all
    select * from {{ ref('stg_tiktok__customers') }}
),

enriched as (
    select
        o.order_id,
        o.order_at,
        o.gross_amount,
        o.discount,
        o.shipping_fee,
        (o.gross_amount - o.discount) as net_amount,

        -- Customer dimensions
        c.customer_id,
        c.customer_name,
        c.email,
        c.phone,
        c.city,
        c.province,
        c.country,

        -- Customer segment
        case
            when c.total_orders >= 10 then 'VIP'
            when c.total_orders >= 5 then 'LOYAL'
            when c.total_orders >= 2 then 'REGULAR'
            else 'NEW'
        end as customer_segment,

        -- Order metadata
        o.source,
        o.status,
        o.payment_type,
        o.loaded_at

    from orders o
    left join customers c
        on o.customer_id = c.customer_id
)

select * from enriched

Join Best Practices

✓ DO

  • Left join để giữ tất cả orders
  • Join on business keys (customer_id)
  • Validate join cardinality (1:1, 1:N)
  • Test với data sample trước

DON'T

  • Inner join (làm mất data)
  • Join on multiple conditions phức tạp
  • Assume 1:1 relationship without checking
  • Join large tables without filtering

📈 Revenue Aggregation

Daily Revenue Metrics

-- models/intermediate/int_revenue__daily.sql

{{
    config(
        materialized='table',
        schema='intermediate',
        tags=['revenue', 'daily']
    )
}}

with orders as (
    select * from {{ ref('int_orders__enriched') }}
),

daily_metrics as (
    select
        date_trunc('day', order_at::timestamp)::date as order_date,
        source,

        -- Volume metrics
        count(distinct order_id) as order_count,
        count(distinct customer_id) as unique_customers,

        -- Revenue metrics
        sum(gross_amount) as gross_revenue,
        sum(discount) as total_discounts,
        sum(shipping_fee) as total_shipping,
        sum(gross_amount - discount) as net_revenue,

        -- Derived metrics
        avg(gross_amount) as avg_order_value,
        sum(gross_amount) / count(distinct customer_id) as revenue_per_customer

    from orders
    where status not in ('CANCELLED', 'RETURNED')
    group by 1, 2
)

select * from daily_metrics
order by order_date desc, source

Key Metrics Definitions

Metric Formula Business Purpose
Gross Revenue SUM(total_amount) Total sales before discounts
Net Revenue Gross - Discounts Actual revenue after promotions
AOV Revenue / Orders Average order value
RPC Revenue / Customers Revenue per customer

💡 Pro Tips

  • Exclude CANCELLED/RETURNED orders khỏi revenue calculations
  • Track metrics by source (Shopee vs TikTok) để compare performance
  • Use date_trunc để aggregate by day/week/month
  • Document metric definitions trong schema.yml

Bonus Challenge

-- Tính Customer Lifetime Value (LTV)
-- models/intermediate/int_customer__ltv.sql

{{
    config(materialized='table')
}}

with orders as (
    select * from {{ ref('int_orders__enriched') }}
),

customer_metrics as (
    select
        customer_id,
        count(distinct order_id) as total_orders,
        sum(gross_amount) as total_revenue,
        min(order_at) as first_order_date,
        max(order_at) as last_order_date,

        -- LTV calculation
        total_revenue / count(distinct order_id) as avg_order_value,

        -- Days between orders
        max(order_at) - min(order_at) as customer_tenure_days

    from orders
    group by customer_id
)

select * from customer_metrics
