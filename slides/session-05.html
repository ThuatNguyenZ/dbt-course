🤖 Auto-Generated Documentation

DBT tự động generate documentation

✓ Không cần viết tay!

DBT tự động thu thập thông tin từ project của bạn và tạo documentation website.

Commands cơ bản

# Generate documentation
dbt docs generate

# Serve locally (default port 8080)
dbt docs serve

# Serve trên port custom
dbt docs serve --port 3000

# Generate cho production target
dbt docs generate --target prod

📝 Writing YAML Descriptions

Schema.yml Structure

# models/staging/schema.yml
version: 2

models:
  - name: stg_shopee__orders
    description: |
      **Staging model for Shopee orders.**

      This model cleans and standardizes raw order data from Shopee API.
      It includes order identifiers, timestamps, amounts, and status.

      **Source:** Shopee Open Platform API
      **Refresh:** Daily at 2:00 AM UTC
      **Grain:** One row per order

    columns:
      - name: order_id
        description: "Unique identifier for each order"
        tests:
          - unique
          - not_null

      - name: customer_id
        description: "ID of the customer who placed the order"
        tests:
          - not_null

      - name: order_at
        description: |
          Timestamp when the order was placed.

          Converted from UTC to ICT timezone (UTC+7).

      - name: gross_amount
        description: |
          Total order amount before discounts and shipping.

          **Formula:** `total_amount` from Shopee API
          **Currency:** VND

      - name: status
        description: |
          Current order status.

          Possible values:
          - `PENDING` - Order placed but not paid
          - `PAID` - Payment confirmed
          - `SHIPPED` - Package with carrier
          - `DELIVERED` - Customer received order
          - `CANCELLED` - Order cancelled

Documentation Best Practices

📋 Be Specific

Bad: "Order amount"

Good: "Total order amount in VND before discounts and shipping fees"

📐 Include Grain

Example: "One row per order per day"

Helps users understand aggregation level

📖 Use Markdown

DBT supports markdown in descriptions

Use `**bold**`, `*italics*`, code blocks

🔗 Add Context

Include source system, refresh schedule, business logic

⚠️ Common Mistakes

  • Empty or generic descriptions ("This is a table")
  • Not documenting business logic
  • Forgetting to update docs when models change
  • Only documenting columns, not models

🌐 Hosting Documentation

Options để host DBT docs

Method Cost Setup Complexity Best For dbt docs serve Free ✓ Easy Local development S3 + CloudFront ~$1-5/month ✓ Medium Production, team access GitHub Pages Free ✓ Easy Open source projects DBT Cloud $50+/month ✓ Easy Teams using DBT Cloud

Deploy to S3 + CloudFront

  1. Generate documentation 
    dbt docs generate --target prod
  2. Create S3 bucket 
    aws s3 mb s3://your-company-dbt-docs
aws s3 website s3://your-company-dbt-docs/ \
    --index-document index.html \
    --error-document index.html
  3. Upload docs to S3 
    aws s3 sync target/ s3://your-company-dbt-docs/ \
    --exclude "*" \
    --include "catalog.json" \
    --include "manifest.json" \
    --include "index.html"
  4. Setup CloudFront (optional but recommended)
    Use CloudFront distribution for HTTPS and caching

🚀 Automate with GitHub Actions

# .github/workflows/docs.yml
name: Deploy DBT Docs

on:
  push:
    branches: [main]

jobs:
  deploy-docs:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v3
      - run: dbt docs generate
      - run: aws s3 sync target/ s3://your-bucket/

📝 Hands-on Exercise

Bài tập thực hành

Document toàn bộ DBT project

📋 Requirements

  1. Write descriptions cho TẤT CẢ models trong staging/ folder
  2. Write descriptions cho TẤT CẢ columns với:
    • Business definition
    • Data type
    • Any transformations applied
  3. Document sources với:
    • Source system description
    • Load frequency
    • Freshness expectations
  4. Generate docs và review trong browser
  5. Deploy docs lên S3 (bonus)
